Pertanyaan Refleksi
Setelah menyelesaikan kode, jawab pertanyaan berikut:

1. Apa fungsi keyword 'extends' dalam deklarasi class Java?
2. Jelaskan perbedaan antara Method Overloading (yang ada di latihan sebelumnya) dengan Method Overriding (yang ada di latihan ini)!
3. Apa kegunaan keyword 'super' di dalam constructor subclass (Manajer)?
4. Jika variabel di class Karyawan diubah access modifier-nya menjadi 'private', apa yang akan terjadi pada class Manajer? Bagaimana solusinya?
5. Mengapa kita perlu menggunakan Inheritance? Apa keuntungan utamanya dibandingkan membuat dua class yang terpisah sama sekali (misal class Karyawan dan class Manajer dibuat terpisah tanpa hubungan)?

Jawaban:

1. extends dipakai ketika sebuah class ingin mewarisi sifat atau perilaku dari class lain. Dengan kata lain, lewat extends, sebuah class baru bisa langsung memakai variabel dan method yang sudah dimiliki class induknya tanpa perlu menulis ulang. Jadi class child tinggal menambah atau memodifikasi fitur yang dibutuhkan.
2. Method Overloading itu membuat beberapa method dengan nama yang sama di dalam class yang sama, tapi bedanya ada di jumlah parameter atau tipe datanya. Tujuannya biar satu method bisa dipanggil dengan berbagai bentuk input.
   Sementara Method Overriding itu mengganti ulang isi method yang diwariskan dari parent di dalam class child. Namanya tetap sama, parameternya sama, tapi isinya dibuat ulang supaya perilakunya lebih sesuai dengan kebutuhan class anak.
3. super digunakan untuk memanggil constructor milik class induk. Fungsinya supaya variabel-variabel yang dimiliki parent (seperti nama dan gajiPokok) bisa langsung diisi saat objek Manajer dibuat, tanpa harus mengulang deklarasi atau inisialisasinya di class Manajer.
4. Kalau variabel parent memakai private, class Manajer tidak akan bisa mengaksesnya secara langsung. Karena private benar-benar hanya bisa dipakai di dalam class itu sendiri.
   Solusinya:
   Ubah access modifier menjadi protected supaya child bisa mengakses, atau biarkan private, tapi dibuat getter/setter sehingga Manajer tetap bisa mengambil nilainya lewat method.
5. Inheritance dipakai supaya kita tidak perlu menulis ulang kode yang sama. Hal-hal umum disimpan di parent, dan anak tinggal menambah bagian yang khusus. Keuntungannya: kode lebih rapi, mudah dirawat, dan pengembangan jadi lebih cepat.
